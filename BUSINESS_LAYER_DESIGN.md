# Business Layer Design

The project now contains business-layer code for user authentication, user management, personal profile updates, and animal rescue management.

## UserService

File: `src/main/java/com/animalsalvation/service/UserService.java`

Main responsibilities:

- Login validation: `login(username, password)`
- User management: `addUser(...)`, `removeUser(...)`, `updateUser(...)`, `findAllUsers()`
- Password management: `changePassword(...)`, `resetPassword(...)`
- Personal profile update: `updatePersonalInfo(...)`
- User persistence: `saveUsers(...)`, `loadUsers(...)`

Default account:

- Username: `op`
- Password: `666666`
- Role: `ADMIN`

## Role-Based UI

File: `src/main/java/com/animalsalvation/controller/MainController.java`

The main UI now starts from a login page. After login, the menu is generated by role:

- `ADMIN`: user management, animal information, rescue tasks, adoption applications, search/sort, operation history, personal profile.
- `STAFF`: animal information, rescue tasks, search/sort, personal profile.
- `ADOPTER`: adoption applications, animal search, personal profile.

The UI uses a muted green/gray/brown palette with custom button, panel, menu, input, and table styling instead of the plain JavaFX default look.

## User Data Model

File: `src/main/java/com/animalsalvation/entity/User.java`

Fields:

- `id`
- `username`
- `passwordHash`
- `realName`
- `phone`
- `email`
- `role`
- `enabled`
- `createTime`
- `lastLoginTime`

## User File Storage

File: `src/main/java/com/animalsalvation/storage/UserFileStorage.java`

Default file:

```text
data/users.csv
```

The file stores user data in UTF-8 CSV format. Passwords are stored as SHA-256 hashes, not plain text.

## Existing Other Business Modules

File: `src/main/java/com/animalsalvation/service/AnimalService.java`

This service already contains the main animal rescue business modules:

- Animal information management
- Rescue task management
- Adoption application management
- Operation history and undo
- Hash search, binary search, binary search tree search
- Sorting and traversal
- All business data file saving/loading

So the project business layer now includes both user-related modules and animal rescue workflow modules.
